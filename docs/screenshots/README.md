# Screenshots — admin console

A visual tour of the **rustdesk-api** admin console. All data below is **fictional demo data**
generated by [`DemoShowcaseSeeder`](../../database/seeders/DemoShowcaseSeeder.php) (example.com
hosts, RFC-5737 IPs) — no real devices or users.

> Independent, third-party open-source project — **not affiliated with or endorsed by RustDesk.**

## Dashboard
Device / user counts, a 24-hour sessions trend, a 14-day connections + new-devices chart, and a
recent-devices list.

![Dashboard](dashboard.png)

## Devices
Searchable inventory with owner, device group, strategy, OS, version and live online status —
plus bulk-assign of owner / group / strategy.

![Devices](devices.png)

## Strategy (Security-Settings) editor
The signature feature: a Strategy editor laid out like the RustDesk client's **Settings**
(General / Security / Network), with tri-state toggles, "set all", custom keys, and
device / user / group assignment — pushed to clients over the heartbeat.

![Strategy editor](strategy-editor.png)

## Address books
Personal and **shared / team** books, each with peer and tag counts. Opening a book gives the
RustDesk-client-style manager (peer cards, colored tags, add/edit dialogs).

![Address books](address-books.png)

## Connection logs (audit)
Connection audit with the **RustDesk 1.4.9** additions — an **Auth** column (how the session
authenticated: click / one-time / permanent password + TOTP / trusted device) and a
controller-attribution indicator.

![Connection logs](connection-logs.png)

## Webhooks / notifications
Push alarms and connection / device events to **Slack**, **Telegram**, or a generic
**HMAC-signed** JSON endpoint, with per-hook test + delivery status.

![Webhooks](webhooks.png)

## Alarms
Security / connection alarms (non-whitelisted IP, session-scope violation, excessive logins,
new-connection with auth details).

![Alarms](alarms.png)

## Live sessions
Currently-connected devices with force-disconnect.

![Live sessions](sessions.png)

## API keys
Scoped keys for the admin REST API (`/api/v1`).

![API keys](api-keys.png)

## Client config generator
Produce a server-config string, mobile **QR**, per-OS `--config` command, and renamed-installer
filename to roll out pre-configured clients.

![Client config](client-config.png)

---

### Regenerating these screenshots

```bash
# Inside the toolchain image, boot the app on SQLite with demo data and capture:
docker run --rm -v "$PWD":/app -w /app rustdesk-api-php-toolchain bash docker/demo-shots.sh
```

The capture spec is [`e2e/screenshots.spec.ts`](../../e2e/screenshots.spec.ts); it is excluded
from the CI gate.
